# 사고보고 — Wi-Fi 경유 침투 → CAN 인젝션으로 구동계 변조

- **시나리오**: `b3_can` · 계층 ④⑧ 내부버스
- **위협**: CAN 인젝션 (내부버스 구동계 변조) (무결성 — 무인증 버스 악용)
- **프레임워크 매핑**: MITRE T0859 (Valid Accounts), T0814 (DoS), T1565 (Data Manipulation) / SPARTA -
- **탐지기**: 경량 MLP(윈도 특징) · 점수 1.0 · 탐지지연 -

## 증거
| 지표 | 값 |
| --- | --- |
| 미등록ID비율 | 0.25 |

| 공격유형(예측) | 퍼지 |

## 규칙 근거(설명가능)
- 미등록 CAN ID 비율 25% — 퍼지/주입 정황
- 최소 IAT 0.015 ms — 버스 폭주(DoS) 정황

## 위협 분석(에이전트)
**위협 분석 결과: 높은 신뢰도의 CAN 버스 침투 공격 탐지**

Wi-Fi 침투 이후 CAN 버스에서 미등록 ID 25%와 극도로 짧은 IAT(0.015ms)가 동시 관찰되었으며, 이는 정상 차량 통신 패턴과 완전히 상이한 특성입니다. 특히 단일 ID 점유율(12%)과 고유 ID 수(23개) 조합에서 비정상적 버스 폭주 및 구동계 명령 주입이 진행 중일 가능성이 높습니다. 경량 MLP 모델의 1.0 공격확률은 이러한 다층적 이상 신호들을 종합적으로 포착한 결과로, 즉각적인 CAN 격리 및 ECU 상태 검증이 필요합니다.

## 대응 플레이북 (NIST CSF)
### 대응(Respond)
- 안전모드 진입 → 차량 정지(구동 토픽 차단)
- 침해 세그먼트(도메인) 격리
- 수동 통제로 전환
### 복구(Recover)
- CAN 트레이스 포렌식 → 악성 ID·주입 프레임 식별
- ECU 펌웨어 무결성 검증 + 키 로테이션
### 예방·파훼(Protect)
- CAN MAC(메시지 인증) 추가
- ID 화이트리스트·주기(period) 검사 상시
- 구동계·센서 도메인 분리 + IDS 상주

## 지휘 요약(에이전트)
# CAN 인젝션 위협 대응 요약

**위협상황**: 무인증 CAN 버스 악용으로 구동계 명령 변조 위험(T0859/T1565). 즉시 안전모드 진입 후 구동 토픽을 차단하여 차량을 물리적으로 정지시키고, 침해된 내부버스 세그먼트(도메인 ④⑧)를 격리합니다. **우선순위**: ① 즉시조치(1단계: 차량정지)→ ② 복구(격리된 세그먼트 무결성 검증, CAN 메시지 무결성 체크)→ ③ 근본대응(버스 인증메커니즘 추가, 메시지 암호화/서명 강제화, 비정상 CAN 프레임 탐지 시스템 구축).
